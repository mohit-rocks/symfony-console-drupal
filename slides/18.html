  • First of all, let’s see why we need Console commands considering we already have Drush.
  • The main intention is:
    • To provide a single entry point for which users can run the command
    • More aligned to Symfony eco-system which we’ve been adopting for quite some time
    • Having another tool in the arsenal is a good thing
    • We can also convert existing core/scripts/drupal application to Console commands
      • This application is alredy powered by console
      • However, if we have something inbuilt in core, we can convert those to individual commands
      • We can even ship commands in Drupal core itself, like Symfony does
      • One use case can be, we can have recipe related commands that are out of the box in Drupal core itself
    • It has additional features like various formatting options like table helper, questions helper, etc for various formatting options.
      • Apart from that, it has multiple components to format output like
      • Title
      • Section
      • Listing
      • Comment
      • Table etc
    • Since commands are services you can run service from regular code like call the service directly etc.
      • Allows to run certain commands from UI
